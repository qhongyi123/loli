# 萝莉公司（Loli Company）

> 一个以病毒传播为核心玩法的前端网页小游戏。
> 本文档用于指导后续 AI 助手沿袭已有风格进行开发。

---

## 一、游戏概述

玩家扮演病毒的操纵者，选择一个病毒和一个世界，进入该世界后通过传播病毒获取进化点，用于升级病毒。同时，该世界的人类会研发解药，媒体会根据病毒爆发情况发布各类新闻。每个病毒有各自独特的终极目标——目标可能是杀死全人类，也可能是达成某种特殊的转化。

### 核心循环

1. **选择病毒** → 2. **选择世界** → 3. **传播病毒获取进化点** → 4. **升级病毒** → 5. **应对人类解药研发** → 6. **达成病毒专属目标**

## 二、美术风格

### 2.1 总体定位：**复古印刷 / 野兽派平面设计（Brutalist Print）**

整体视觉风格以"老式海报 / 工业印刷品 / 地下出版物"为基调，强调高对比度、几何碎片和粗糙质感。

### 2.2 色彩体系

| 角色 | 色值 | 用途 |
|------|------|------|
| 主红 | `#C41E3A` | 主要强调色，按钮、条带、三角形 |
| 深红 | `#6B1020` | 次要红色块 |
| 黑 | `#151515` | 边框、文字、几何块 |
| 米白 | `#F3EDE1` | 全局背景底色 |
| 金 | `#B8924E` | 点缀装饰 |
| 深金 | `#7A5F30` | 辅助金色 |

- 整个界面以米白为底，黑、红为主色调，金色仅作点缀。
- 杜绝渐变、荧光色、霓虹、高饱和糖果色。

### 2.3 图形语言

- **几何碎片**：大量使用三角形、细长条、菱形、空心圆等几何元素散布在背景中，营造"碎纸片 / 印刷裁切"感。
- **斜向切割**：红色、黑色、金色的对角斜条带（`struct-slice`）横跨画面，形成第一层结构。
- **磨损纹理**：多层 SVG 噪点叠加（纸纹、老化斑点、划痕），模仿旧纸张/旧印刷品质感。
- **印章印记**：画面中散布红色/黑色边框小印章（如`LOLI`），倾斜放置。
- **放射线**：画面角落有放射状细线簇，模拟蓝图/设计稿的辅助线。
- **卡片倾斜**：主菜单卡片呈旋转错位排列，hover 时上浮并改变投影色。

### 2.4 字体体系

- 首选：**思源黑体（Source Han Sans SC）**，备选 Noto Sans SC、微软雅黑。
- 所有文字使用粗体/超粗体（`font-weight: 800-900`）。
- 标题使用超大间距（`letter-spacing: 6-16px`）。
- 不使用衬线体、手写体、装饰字体。

### 2.5 动效原则

- 过渡动画应**干脆利落**，无弹性/缓出/弹簧效果。
- Hover 效果以颜色反转、平移位移为主（如按钮 hover 时红底变黑底、红字）。
- 加载动画使用机械感的时钟指针旋转。
- 不使用淡入淡出外的复杂过渡（fade、glow、pulse 等均不适用）。

### 2.6 尽可能少使用的视觉元素

- 渐变（linear-gradient 仅用于纹理模拟，不可用于 UI 装饰）
- 圆角（除少数特定几何元素如空心圆外）
- 毛玻璃 / 模糊效果
- 阴影扩散（仅允许硬边投影 `box-shadow` with 0 blur）
- 霓虹发光
- Emoji 作为 UI 元素

## 三、界面结构

### 现有页面

| 页面 | ID | 说明 |
|------|-----|------|
| 加载画面 | `screen-loading` | 时钟动画 + 随机小贴士 |
| 开始画面 | `screen-start` | 标题 + START 按钮 + 设置入口 |
| 选择画面 | `screen-select` | 4 张卡片（开始游戏/继续游戏/自定义病毒/创意工坊）+ 翻页三角 |
| 设置面板 | `settings-overlay` | 深色遮罩 + 弹窗面板，当前仅有"过渡动画"开关 |

### 未来的游戏主界面（待开发）

主界面需保留上述全部美术风格要素（色彩、几何碎片、纹理、斜切条带）。布局上可参考以下面板结构：

- 病毒面板
- 进化升级树 / 按钮区
- 解药研发进度条

- 事件日志

所有面板均遵循现有风格：硬边框、方正造型、米白底色、黑/红强调。

## 四、AI 助手开发指引

### 4.1 你需要遵循的规则

1. **沿用已有美术风格**：所有新增 UI 元素必须继承 `style.css` 中定义的色彩、几何图形、纹理、动效规范。不可自行引入新的视觉语言。
2. **保持代码风格一致**：继续使用原生 HTML + CSS + JS，CSS 变量已定义在 `.game-frame` 上，新增样式应优先使用这些变量。
3. **每次只根据用户当前指令完成一个任务**：不要自行扩展需求范围，不要主动重构已有代码。
4. **不要擅自修改基础视觉元素**：`struct-slice`、`geo`、`radial`、`stamp`、`scratches`、`texture-overlay` 等背景层属于全局美术基底，非必要不改动。

### 4.2 项目文件结构

```
萝莉公司游戏界面/
├── index.html          # 主 HTML（页面结构 + 背景层）
├── css/
│   └── style.css       # 全局样式（色彩、纹理、几何、UI 组件）
├── js/
│   └── main.js         # 页面切换、设置、加载逻辑
├── data/
│   └── tips.js         # 加载小贴士数据
└── README.md           # 本文档
```

### 4.3 开发建议

- 新增 JS 可新建文件放入 `js/` 目录，通过 `<script>` 标签引入。
- 图片资源暂用占位符，后续用户会提供实际素材路径。
- 如需存储游戏数据，使用 `localStorage`，key 统一加前缀 `loli_company_`。
